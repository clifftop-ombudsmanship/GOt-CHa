I've been experimenting with and learning Git, using a few different version control Git-repository hosting services. My very first encounter with Git was GitHub, when I was researching conceptual modeling and semantic metadata registration. The International Federation of Library Associations and Institutions (IFLA) had GitHub repositories, and I just wandered into the environment wondering what it is.

Then, as I investigated further, I found GitLab, and Bitbucket, and Azure DevOps, and finally Git itself. Now I'm being creative in a way that allows me to use a combination of tools and sites, which I really like. Referata is another hosting service I'm using, my favorite wiki ever, and it will be my interface to reconnect with FANDOM, which is what I was using before I stepped into the world of metadata and version control.

I appreciate some different aspects of each of these sites I've mentioned, and so have a number of working styles going on. I'm going to use Azure DevOps for some more private work with the name Zuoken in connection with the name Geshtai, both of which are Dungeons & Dragons Product Identities and therefore unavailable for my public use. However, GitHub has a great deal with its offering of Pages, and so I will maintain a few public Pages with my repositories, “GOt-CHa” being the public partner to my private “Geshtai.”
